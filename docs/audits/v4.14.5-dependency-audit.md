# Dependency vulnerability audit — v4.14.5

Date: 2026-07-09 · Branch: `chore/audit-fix` · Source: `npm audit --json`

## Summary

`npm audit` reports **12 advisory entries** (2 high, 10 moderate). These roll up
to **7 distinct advisories** — matching GitHub Dependabot's count of 7 (Dependabot
dedupes by advisory; `npm audit` counts once per affected package in the tree).
Dependabot's severity split (2 high / 3 moderate / 2 low) is the per-advisory
view; `npm audit` promotes the two low undici advisories to moderate at the
package level via the dependency chain.

**Category (a) — safe to fix now: NONE.** `npm audit fix` **without `--force`**
is a genuine no-op here (zero drift to `package.json` / `package-lock.json`).
Every root requires a breaking major bump, so nothing was changed in this slice.

**Before / after advisory count: 12 → 12** (no non-breaking fix available).

No `.npmrc` audit-level, no `audit-ci` allowlist, no ignore rule was added.
The blocked items are documented below rather than suppressed.

## Fix availability & categorization

### (a) Safe to fix now — **empty**
`npm audit fix` (no `--force`) applied zero changes. `npm audit`'s optimistic
`fixAvailable: true` on `request` / `request-promise-core` / `@cypress/request-promise`
cannot actually be realized without a major bump of the direct parent that pins
them, so no non-breaking fix is applicable.

### (b) Needs a major (breaking) bump — reported, NOT applied
| Direct dependency | Current | Suggested fix | Clears |
|---|---|---|---|
| `nodemailer` | `^8.0.11` | `9.0.3` (major) | #1 |
| `discord.js` | `^14.25.1` | `13.17.1` (major **downgrade** — loses the v14 API surface) | #2, #3, #6, #7 (undici) |
| `dockerode` | `^4.0.9` | `5.0.1` (major) | #5 (uuid via dockerode) |
| `node-telegram-bot-api` | `0.67.0` | `1.1.2` (major) | #4 (request/SSRF) + #5 (uuid via @cypress/request) |
| `uuid` | `^9.0.0` | `14.0.1` (major; `11.1.1`+ is the minimum patched) | #5 (direct uuid) |

Each is a breaking bump across a major boundary and must be validated on its own
(API changes in nodemailer 8→9, dockerode 4→5, node-telegram-bot-api 0.67→1.x,
uuid 9→14). Not taken here per the "no breaking bump without reporting first"
rule.

### (c) No clean upstream fix — needs a workaround / accepted risk
- **`request` / `@cypress/request` / `request-promise-core`** — deprecated and
  unmaintained; no patched version exists. They are removed only by bumping
  `node-telegram-bot-api` to `1.x` (item b), which drops `@cypress/request`.
- **`undici` under `discord.js@14`** — no `discord.js` 14.x release ships a
  patched `undici`. Two options: (i) the suggested downgrade to `discord.js@13`
  (item b, loses v14 features), or (ii) an npm `overrides` pin forcing `undici`
  to a patched build under `discord.js` — a manual pin that risks breaking
  discord.js's internal undici usage and must be behind a test pass. Neither is
  applied here.

## Recommended follow-up (separate slice, not this one)
Bump the (b) items one dependency per PR, each behind the full suite:
`nodemailer@9`, `dockerode@5`, `node-telegram-bot-api@1.x`, `uuid@11+`. Evaluate
the `discord.js` undici question (v13 downgrade vs. an `overrides` pin) on its
own, since it is the only one with a meaningful behavioural trade-off.

## Residual risk accepted for now
The two HIGH advisories and the uuid MODERATE are **not reachable** in Aiden's
call paths (see the table). The four undici advisories are only partially
reachable (they require a hostile response from Discord's official API). No
change to `package.json`/`package-lock.json` was made; the fixes are queued as
breaking bumps for a dedicated slice rather than forced in unverified.
